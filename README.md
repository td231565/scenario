# Scenario
紀錄實際遇到的情境問題和解決方法


**取得資料錯誤時，如何引導用戶下一步**
- 開放式：在用戶能導覽的入口網站中，簡單顯示錯誤訊息，讓用戶可以進行其他操作。

  如選擇其他分類、看其他文章。
  
- 封閉式：在特定功能頁面中，跳出錯誤訊息彈窗，並用按鈕引導用戶下一步操作，避免用戶失去焦點而跳出。
  
  像是抽獎活動頁面取得資料失敗，留在原畫面的話，用戶需要自己去想接下來怎麼辦，在手機裝置很容易被滑掉 (跳出)。


**用所見及所得編輯器處理 HTML**
Q：有的編輯器支援 HTML 模式，但直接貼上 HTML 再預覽會跑版

A：預覽模式會將 HTML 當成內文預處理，會被加入一些 <p>, 空格符文等，導致 HTML tag 被破壞

遇到有這樣問題的文字編輯器，只能貼上後直接儲存，不要轉換成預覽模式觸發預處理
